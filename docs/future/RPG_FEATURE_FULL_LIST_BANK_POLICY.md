# RPG Feature Full List Bank Policy

このファイルは、将来実装したい機能をBank 0に入れず、どのBankへ逃がすかを固定するためのメモです。

## 絶対ルール

- 新規ゲーム機能は基本的にBank 0へ入れない。
- main.cへ大型処理を増やさない。
- 大量データはROM bankへ置く。
- WRAMには現在状態と一時状態だけを置く。
- VRAM/OAMを消費する機能は、scene単位でロード/解放する。

## 優先度

1. Bank 0 overflowを消す
2. ダンジョン・町・遺跡の安定化
3. 戦闘中敵BG表示の安定化
4. アイテム/装備UIの安定化
5. 戦闘後成長、熟練、ひらめきの最小実装
6. 敵ランクとエンカウントテーブル
7. イベントVM/長編クエスト
8. 5000x5000相当フィールドのチャンク化
