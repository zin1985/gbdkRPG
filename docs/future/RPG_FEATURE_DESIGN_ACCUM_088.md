# RPG将来機能検討 蓄積メモ rpg088

## 今回取り込んだ検討結果

### 1. MP消費制
- `とくぎ` と `かいふく` は使用回数制ではなく、MP消費を主軸にする。
- 既存の `skill_uses` / `heal_uses` は削除または未使用化し、UI上の `M` と仕様を一致させる。
- 初期コストは以下とする。
  - とくぎ: MP 2
  - かいふく: MP 4
- MP不足時は行動失敗扱いにし、プレイヤーターンへ戻す。
- HP満タン時の回復はMPを消費しない。

### 2. SkillDefテーブル方式
- 固定4コマンドは維持する。
- 内部的には `SkillId` / `SkillKind` / `TargetType` / `SkillDef` を導入し、将来の複数技へ拡張しやすくする。
- 今回は技一覧UIは追加しない。
- 主人公の `とくぎ` は `SKILL_SLASH`、`かいふく` は `SKILL_HEAL_SIMPLE` に固定する。

### 3. 効果計算式
- 通常攻撃: `attack * 2 - defense`
- とくぎ: `skill * 3 + attack + power - defense / 2`
- かいふく: `heal * 2 + power`
- ダメージ計算は `INT16` を経由し、最低1ダメージを保証する。
- HP/MP上限999に合わせ、最終ダメージは999でクランプする。

### 4. Bank 0対策
- 前回ビルドログで `ROM_0` がほぼ満杯になり、`romusage` が _HOME overflow を警告した。
- 今回は成長メッセージ文字列を `messages_bank.c` へ移し、`message_show()` 経由に変更した。
- これにより、追加実装しながらBank 0増加を抑える方針とした。

## まだ取り込んでいない検討結果
- 複数キャラ成長
- GrowthResult配列化
- 戦闘メッセージキュー
- 優先度付きログ
- ターン/行動単位グルーピング
- 演出同期 `fx_type`
- A長押し高速送り / Bスキップの全体共通化
- 状態異常システム
- 装備/アイテム/セーブデータ統合
