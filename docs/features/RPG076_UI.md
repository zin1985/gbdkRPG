# rpg076_ui

## 修正概要
rpg075でまだ発生していた、戦闘1ターン目のコマンド選択時の画面乱れをさらに抑える修正。

主な対策は以下。

1. 戦闘開始メッセージ表示前は、コマンドUIを描かず、空の戦闘BG＋敵スプライトだけにした。
2. メッセージ終了後に、Windowを非表示にするだけでなく `move_win(JP_WIN_X, 144u)` で画面外へ退避する。
3. 初回コマンド前に、BGスクロールを0へ戻し、フォントキャッシュを再初期化し、BGを全消ししてからUIを再描画する。
4. 左上パーティ枠を15タイル幅から20タイル幅へ変更した。
5. rpg074の2人目HP/MP表示が右枠線を上書きしていたため、枠幅と座標を見直した。

## 注意
`そうりょ` はまだ表示だけの仮枠。戦闘参加、actor追加、OAM追加はしていない。
