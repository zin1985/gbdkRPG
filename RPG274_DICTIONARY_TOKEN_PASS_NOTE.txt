# RPG274 Dictionary / Token Message Pass

対応内容:

1. 道具名・装備名の名前付きトークン化
   - `menu_text_dict.c/.h` を追加。
   - 道具IDから辞書化された名前を展開する `menu_dict_item_name()` を追加。
   - 装備名は `party_runtime.c` 内の重複文字列を外し、道具ID名を共通参照するよう変更。

2. 店・戦闘・メニュー定型文の辞書化土台
   - `MENU_DICT_MSG_*` を追加。
   - もちもの画面の定型文、効果なし、対象選択、空表示などを辞書経由に変更。
   - 戦闘報酬のドロップ名は `inventory_get_item_name()` 経由のため辞書名を使う。

3. メッセージビルド用スクリプト
   - `tools/build_message_dict.py` を追加。
   - 人間は `{ITEM_HERB}` のような名前付きトークンで書き、ビルド時に数値トークン列へ変換する方針。

4. ROMには圧縮済みトークン列を入れる
   - `menu_text_dict.c` 内の文字列は `0x1F + token_id` 形式の圧縮トークンを含む。
   - 共有語彙は「の」「けん」「つえ」「ゆみ」「よろい」「ポーション」「まほう」「種」「指輪」など。

5. 道具一覧では辞書展開後に既存の行タイルキャッシュへ格納
   - 既存のFF4式 `g_inventory_row_tiles` を活かす。
   - 新規行だけ辞書名を展開し、タイル列化してキャッシュ。
   - 一覧本体と下部説明文の分離方針を維持。

6. 会話本文への拡大
   - 今回は土台とスクリプトまで。
   - いきなり全会話へ適用せず、店・戦闘・メニュー定型文から拡大する前提。

追加で今回、`shop_runtime.c`、`battle_command_ui_runtime.c`、`menu_runtime.c` の一部固定ラベルも `menu_dict_message()` 経由に変更済み。

注意:
- `menu_text_dict.c` は bank 25。Bank 0へ単語表を置かない。
- 生成後は `inventory.rel`, `party_runtime.rel`, `menu_text_dict.rel` を再コンパイルする。
- 実機確認では、もちものスクロール後に下部説明文が壊れないかを重点確認。


確認ポイント:
- もちもの一覧の初回表示、1行クロール、下部説明文字の文字化け有無。
- 装備画面で未装備スロットが「なし」と表示されること。
- 店、宿屋、戦闘コマンド、メニューラベルが表示されること。
- Bank 25 が増えるため、もしBank 25が詰まる場合は辞書を別Bankへ移す。
