# rpg195_sprite_cleanup_battle_item_pages_fullwidth

## 対応内容
- パラメーター画面で、移動中の勇者/味方スプライトが残る問題を対策。
  - パラメーター画面描画前にOAM 40個をすべて退避。
  - 1ページ目のみキャラスプライトを再表示。
  - 2ページ目/3ページ目や終了時はOAMを退避して残骸を残さない。
- 戦闘中に「どうぐ」を開いた時、味方スプライトがメニュー上に表示される問題を対策。
  - `player_battle_item()` で道具メニュー表示前に `hide_all_sprites_safe()` と `HIDE_SPRITES` を実行。
  - 道具メニュー終了後は `battle_enter_render_once()` で戦闘画面を再構築。
- 戦闘中どうぐは、既存の2列/ページ式道具欄をそのまま使用。
  - 上下左右カーソル移動。
  - 下端で次ページ、上端で前ページ。
  - `▲/▼` 表示も共通。
- 数字と英字を全角表示に戻し。
  - `utf16_HantoZen()` で ASCII `0-9 / A-Z / a-z` を全角へ変換。
  - `HP/MP/A/B/G` や数値が日本語UIに馴染む見た目へ戻る。
