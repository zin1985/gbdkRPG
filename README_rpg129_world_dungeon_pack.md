# rpg129_world_dungeon_pack

## 目的

rpg128_enemy_bg_tiles をベースに、Bank 0 を増やしすぎない方針を維持しつつ、世界を少し広げるための追加実装を行った版です。

## 追加した内容

- フィールド上の行き先を追加
  - 既存の町
  - 新規：港町
  - 新規：洞窟ダンジョン
  - 新規：古代遺跡
- ダンジョン用BGマップチップを追加
  - dungeon wall 4タイル
  - dungeon pit/hazard 4タイル
  - treasure chest 4タイル
- 新しいエリア別マップを `map_data_bank.c` に追加
  - `port_collision16_map` / `port_object16_map`
  - `dungeon_collision16_map` / `dungeon_object16_map`
  - `ruins_collision16_map` / `ruins_object16_map`
- 新しいメッセージを `messages_bank.c` に追加
  - 港町へ入る/到着
  - 洞窟へ入る/到着/退出
  - 古代遺跡へ入る/到着
  - 宝箱取得/空箱
  - 罠床ダメージ
  - 遺跡の石碑風メッセージ
- 検討中機能の一部を軽量実装
  - エリア別ランダムエンカウント率
  - 洞窟/遺跡の危険エリア判定
  - 罠床による小ダメージ
  - 宝箱取得フラグ
  - クエスト開始/進行の簡易接続
  - 町系エリア判定の共通化

## Bank 0保護方針

大量データは以下のbanked側へ寄せています。

- マップ定義：`map_data_bank.c`
- 追加マップチップ：`sprites.c` の既存マップBGタイル列
- メッセージ：`messages_bank.c`

ただし、エリア遷移・イベントルーティングのため、`main.c` には小さな分岐追加があります。
正確なBank 0使用率は、ユーザー環境で `build.sh` 実行後の `romusage` で確認してください。

## 追加エリアの入口

フィールド上の目安：

- 町：既存の `x=14, y=2`
- 港町：`x=14, y=14`
- 洞窟：`x=1, y=14`
- 古代遺跡：`x=1, y=2`

## 確認してほしいこと

1. `./build.sh` でビルドできること
2. `romusage` で Bank 0 overflow が出ないこと
3. フィールドから港町・洞窟・古代遺跡へ入れること
4. 各エリアからフィールドへ戻れること
5. 洞窟/遺跡で敵エンカウントが発生すること
6. 洞窟/遺跡で罠床メッセージと小ダメージが発生すること
7. 宝箱を調べるとポーションが増え、2回目は空箱メッセージになること
8. 3体32x32敵のBG表示が引き続き崩れていないこと
