# CHECK rpg149_target_select_enemy_size_table

## 目的

戦闘時に攻撃対象を選択できるようにし、敵編成テーブルをモンスターサイズ別に整理する。

## 実装内容

### 攻撃対象選択

- `BSTATE_TARGET` を追加。
- 「こうげき」選択後、即攻撃せず対象選択状態へ移行。
- 攻撃系の「とくぎ」も対象選択状態へ移行。
- 回復系の「とくぎ」および「かいふく」は従来通り味方側に即実行。
- 対象選択中の操作:
  - 左 / 上: 前の生存敵へ移動
  - 右 / 下: 次の生存敵へ移動
  - A: 決定して攻撃 / 攻撃技実行
  - B: キャンセルしてコマンド選択へ戻る

### モンスターサイズ

`battle_data.h` にサイズ定義を追加。

- `BATTLE_ENEMY_SIZE_S`: 16x16、最大6体
- `BATTLE_ENEMY_SIZE_M`: 32x32、最大3体
- `BATTLE_ENEMY_SIZE_L`: 32x96、最大1体

### 敵テーブル

`BattleEnemyData` に `size_kind` を追加。
`battle_data_bank.c` の敵定義とエンカウント定義をサイズ別編成に変更。

- S編成: 6体 / 4体 / 3体
- M編成: 3体 / 2体 / 1体
- L編成: 1体

### BG描画

`sprite.c/.h` に以下を追加。

- `battle_enemy_bg_draw_all_sized()`
- `battle_enemy_bg_draw_slot_sized()`

S/M/LをBGタイルとして描き分ける。
敵は引き続きOAMを使わないため、パーティアイコン、カーソル、効果演出用のOAM余力を残す。

## 注意点

- Lサイズは現時点では既存32x32敵素材を縦に3段繰り返す仮表示。
- 専用の32x96ボス絵が入ったら、Lサイズ用タイル配置を差し替える。
- 32x96は現行UIと縦方向に干渉しやすいため、将来的にはLサイズ専用のボス戦UIレイアウトを作るのが安全。
- `prebank_check.py` は通過済みだが、この環境では `lcc` がないためROM実ビルドは未実施。
