# rpg126 enemy32 OAM limit fix check

## 目的
戦闘開始時に3体の32x32敵スプライトを横並び表示した際、右端の敵の右半分が消える問題を修正する。

## 原因
Game Boy実機仕様として、同一スキャンラインに表示できるOBJスプライトは最大10個。
32x32敵を8x16 OBJで表示すると、1体あたり横1段に4スプライト必要になる。
3体を同じY座標に並べると 4 + 4 + 4 = 12 スプライトとなり、10個制限を超えるため、OAM順で後ろにある右端敵の右側スプライトが表示されない。

## 修正内容
- `show_one_battle_enemy_sprite()` を修正。
- 敵が1体または2体のときは従来どおり32x32表示。
- 敵が3体のときは、左端と右端を24x32相当、中央を32x32で表示する。
- 同一ラインのOBJ数を `3 + 4 + 3 = 10` に抑える。
- 未使用列のOAMを `move_sprite(..., 0, 0)` で隠す。
- `hide_battle_enemy_sprites()` を0-23番まで隠すよう修正。
- `BATTLE_ENEMY_SPRITE_Y` を32から40へ変更し、上部ステータス枠の下端と敵スプライトの透明領域が重ならないようにした。

## VBA確認ポイント
1. 戦闘開始時、右端敵の右半分が消えないこと。
2. 3体表示時、敵が横方向に少し細く見えても、表示欠けがないこと。
3. 1体・2体表示時は32x32のまま表示されること。
4. 上部ステータス枠の下端と敵表示領域の境目に、白化・欠け・変な重なりが出ないこと。
5. 戦闘後にフィールドへ戻った際、スプライト残骸が残らないこと。
